Bien, en este parrafo hablaremos de las etiquetas de HEADER, que se usan para poder agrupar links, alguna imagen o logo de nuestra pagina
y lo mas importante, nosotros podemos crear tantas etiquetas de HEADER como queramos, siempre y cuando esta cumpla con 2 reglas basicas :

1. No podemos poner un etiqueta de HEADER dentro de otra etiqueta de HEADER.
2. No podemos poner un etiqueta de HEADER dentro de un etiqueta de FOOTER.

Lo que nos lleva a nuestra siguiente etiqueta de FOOTER la cual se utiliza para colocar el pie de pagina de nuestra Pweb.
Esta etiqueta de FOOTER por lo general contienen información como direcciones, contactos o enlaces a políticas de privacidad,
y se puede usar tantas veces como queramos, siempre y cuando cumpla con 2 reglas basicas :

1. No podemos poner un etiqueta de FOOTER dentro de otra etiqueta de FOOTER.
2. No podemos poner un etiqueta de FOOTER dentro de un etiqueta de HEADER.

Y también está la etiqueta de SECTION la cual va colocada entre HEADER Y FOOTER y se utiliza para poder definir secciones dentro de nuestra web
ç y dentro de cada sección podemos agrupar elementos relacionados como etiquetas.

Pero eso NO es todo... pues también tenemos las etiquetas de ARTICLE las cuales se utilizan para definir contenido independiente y reutilizable en nuestra web.

JEJEJEJ aun no termino, también está DIV que hace la misma funcion que SECTION pero sin semántica específica y va colocada entre HEADER Y SECTION O HEADER Y FOOTER.
Por cierto las etiquetas de DIV se pueden meter unas dentro de otras, pero... devemos evitar cosas como:

1. Poner parrafos dentro de varias etiquetas de DIV, ya que esto no tiene sentido y es un error de maquetación.
2. Poner etiquetas de DIV dentro de otras etiquetas de DIV, ya que esto no tiene sentido y es un error de maquetación.
